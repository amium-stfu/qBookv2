# IMPLEMENTATION HANDOFF

## Goal
Set the `MainWindow` window icon to the Hornet icon asset.

## Scope
Included:
- Configure the Avalonia `MainWindow` icon.
- Ensure the icon file is included as an Avalonia resource in the project that owns `MainWindow`.
- Verify the application builds.

Not included:
- Do not change application branding, title text, logos, or toolbar/header icons.
- Do not redesign UI.
- Do not add new dependencies.
- Do not create a release version or Git tag.

## Tasks
1. Inspect the current working tree and avoid overwriting unrelated user changes.
2. Confirm the intended icon file:
   - The repository currently contains `src/HornetStudio.Editor/EditorIcons/hornet.ico`.
   - No file named `Hornet.icon` was found during planning.
   - If `Hornet.icon` exists in a different untracked or external location, copy or add it only if explicitly available in the workspace.
   - Otherwise use the existing `hornet.ico`.
3. Add the icon asset to the application project resources where `MainWindow` is defined:
   - Prefer adding an `AvaloniaResource` entry in `src/HornetStudio/HornetStudio.csproj`.
   - Keep the path stable and readable, for example referencing the existing editor icon path if Avalonia resource resolution supports it, or placing a copy under the app project if necessary.
4. Set the `Icon` property on `src/HornetStudio/MainWindow.axaml`.
5. If a copied icon asset is needed, keep the file name English and consistent, for example `hornet.ico`.
6. Run verification:
   - `dotnet build HornetStudio.sln --no-restore`
   - If restore is required, run `dotnet build HornetStudio.sln` only if approved or already available offline.
7. Update `CHANGELOG.md` only if this repository records small UI fixes there for unreleased local changes.
8. Provide a final summary with changed files and mention which icon file was used.

## Technical Constraints
- Chat responses must be in German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must remain English.
- Keep the implementation strictly scoped to the window icon.
- Use existing Avalonia project patterns.
- Do not introduce new dependencies.
- Do not modify unrelated icon picker assets or widget icons.
