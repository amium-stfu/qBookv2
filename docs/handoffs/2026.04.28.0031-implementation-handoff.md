# IMPLEMENTATION HANDOFF

## Goal
Show a HornetStudio splash screen with logo, title, and progress bar while the application starts, then close it when `MainWindow` is ready to open.

## Scope
- Included:
  - Add a small Avalonia splash window for startup.
  - Display the existing HornetStudio branding: hornet icon/logo, `HornetStudio` title, and progress bar.
  - Show the splash before creating or showing `MainWindow`.
  - Close the splash after `MainWindow` has been assigned and shown or activated.
  - Keep startup behavior otherwise unchanged.
- Not included:
  - Changing application loading logic beyond splash orchestration.
  - Adding new dependencies.
  - Redesigning `MainWindow`.
  - Adding release/version changes.

## Tasks
1. Inspect `src/HornetStudio/App.axaml.cs` startup flow and keep existing logging, shutdown handling, and exception registration behavior intact.
2. Create a new Avalonia window for the splash screen in `src/HornetStudio/`, for example `SplashScreenWindow.axaml` and `SplashScreenWindow.axaml.cs`.
3. Add the new splash files to `src/HornetStudio/HornetStudio.csproj` because default compile items are disabled.
4. Use existing resources where possible:
   - `avares://HornetStudio/EditorIcons/hornet.ico` for the hornet graphic if suitable.
   - Existing `hornet-studio.svg` only if it renders cleanly for the requested layout; otherwise use text for the title.
5. Style the splash window to match the requested mockup:
   - White background.
   - Thin dark border.
   - Left logo area.
   - Large bold `HornetStudio` title.
   - Horizontal progress bar below the title.
   - No resize chrome if Avalonia supports this cleanly.
6. In `App.OnFrameworkInitializationCompleted()`, show the splash before creating `MainWindow`.
7. Create `MainWindow` and `MainWindowViewModel` as before.
8. Ensure the UI thread gets a chance to render the splash before startup work continues, for example by dispatching the remaining startup sequence asynchronously with `Dispatcher.UIThread.Post` or an equivalent Avalonia pattern.
9. Close the splash after `MainWindow` is ready and visible. Avoid leaving the splash as `desktop.MainWindow`; `MainWindow` must remain the shutdown owner because `ShutdownMode` is `OnMainWindowClose`.
10. If progress is only indeterminate, use an indeterminate progress bar. If the startup sequence exposes clear phases without larger refactoring, optionally set simple values such as initializing, creating main window, showing main window.
11. Run `dotnet build HornetStudio.sln --no-restore`.

## Technical Constraints
- Use Avalonia patterns already present in the project.
- Do not introduce new dependencies.
- Keep the change scoped to startup UI and project file registration.
- Chat and planning are German, but code, comments, XML documentation, UI text, file names, and technical identifiers remain English.
- Public classes and public methods touched by the implementation need XML documentation if missing or outdated.
- Avoid unrelated cleanup, refactoring, or behavior changes.

- `src/HornetStudio.Editor/EditorIcons/hornet-studio.svg`

## Notes
- `App.axaml.cs` currently sets `desktop.ShutdownMode = ShutdownMode.OnMainWindowClose` and then directly assigns `desktop.MainWindow = new MainWindow { DataContext = new ViewModels.MainWindowViewModel() };`.
- The splash should be a temporary window, not the desktop lifetime `MainWindow`.
- Because `HornetStudio.csproj` has `<EnableDefaultCompileItems>false</EnableDefaultCompileItems>`, every new `.axaml.cs` file must be explicitly included.
- Verify that the splash appears long enough to render during startup, especially on fast machines.
