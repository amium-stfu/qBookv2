# IMPLEMENTATION HANDOFF

## Goal
Use the same folder template and directory structure for the generated startup `DefaultLayout` that is already used when creating a new project or folder.

## Scope
Included:
- Update first-start default layout creation in `src/HornetStudio/ViewModels/MainWindowViewModel.cs`.
- Ensure the generated `DefaultLayout` directory contains `Folder.yaml`, `Scripts/`, and `Assets/`.
- Generate `DefaultLayout/Folder.yaml` via the existing folder template path and `BuildNewPageYaml(...)` flow.
- Keep existing best-effort startup behavior.

Not included:
- No changes to template content unless a test or build issue proves it is required.
- No new features, UI changes, new dependencies, or release/version changes.
- No broad refactoring of project creation.

## Tasks
1. Inspect `EnsureDefaultLayout(...)` and replace the inline minimal YAML generation with the existing template-based generation.
2. Reuse `CreatePageDirectoryStructure(directory)` before writing the default layout file so `Scripts/` and `Assets/` exist.
3. Write `BuildNewPageYaml("Default Layout", "DefaultLayout")` to `pagePath` only when `Folder.yaml` does not already exist.
4. Keep the current behavior that an existing `DefaultLayout/Folder.yaml` is not overwritten.
5. Add or update focused tests if a suitable test project exists for `MainWindowViewModel`; otherwise verify with build and document that no matching test harness exists.
6. Run `dotnet build AutomationExplorer.sln --no-restore` or the narrowest available build command.

## Technical Constraints
- Follow existing project patterns in `MainWindowViewModel`.
- Do not introduce new dependencies.
- Keep code, comments, XML documentation, UI text, file names, and technical identifiers in English.
- Chat response must be German.
- Preserve best-effort exception handling in startup default layout creation.
- Avoid unrelated formatting or architecture changes.

## Relevant Files
- `src/HornetStudio/ViewModels/MainWindowViewModel.cs`
- `src/HornetStudio.Editor/Templates/Folder.yaml`
- `src/HornetStudio.Editor/HornetStudio.Editor.csproj`
- `src/HornetStudio/HornetStudio.csproj`

## Notes
- `FolderTemplateRelativePath` points to `Templates\Folder.yaml`, which is copied to output by `HornetStudio.Editor.csproj`.
- New project and folder creation already use `BuildNewPageYaml(...)` and `CreatePageDirectoryStructure(...)`.
- `EnsureDefaultLayout(...)` currently creates only a minimal inline YAML file and does not create `Scripts/` or `Assets/`.
