# IMPLEMENTATION HANDOFF

## Goal
Reorganize the solution so project folders are synchronized with project names and grouped under `src/`.

## Scope
Included:
- Move production project folders from the repository root into `src/`.
- Rename folders so each folder name matches its project name where appropriate.
- Update `AutomationExplorer.sln` project paths.
- Update all affected `ProjectReference` paths.
- Verify build after the move.

Not included:
- Namespace renaming unless required by broken references.
- Functional changes.
- Refactoring unrelated code.
- Adding new dependencies.
- Release versioning.

## Tasks
1. Inspect all project files and project references.
2. Decide the final folder mapping before moving files.
3. Create `src/` if it does not exist.
4. Move each production project into its target folder.
5. Update `AutomationExplorer.sln` paths.
6. Update affected `ProjectReference` entries in `.csproj` files.
7. Check whether `UiEditor` is an actual project, a stale folder, or content that belongs under `src/AutomationExplorer.Editor/`.
8. Update relevant documentation if the repository structure is documented.
9. Run `dotnet build AutomationExplorer.sln --no-restore`.
10. Summarize changed files and any remaining risks.

## Technical Constraints
- Follow existing project patterns and formatting.
- Do not change public APIs unless required.
- Keep code, comments, XML documentation, UI text, file names, and technical identifiers in English.
- Chat responses must be in German.
- Do not add dependencies.
- Keep the change structural only.
