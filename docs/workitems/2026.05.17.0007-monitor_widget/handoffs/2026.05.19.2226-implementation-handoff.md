# IMPLEMENTATION HANDOFF

## Goal
Refine the compact MonitorWidget rule rows so active state is easier to see and row actions are visually lighter:
- remove the dedicated LED as the primary active indicator
- indicate active rules through severity-colored row background/border styling
- make the cog action icon smaller and visually buttonless, while keeping the click/touch target usable

## Scope
- Change only MonitorWidget compact row visuals.
- Keep row content as `EventId`, `EventText`, and row action menu.
- Keep the existing row action menu behavior with `Edit` and `Delete`.
- Keep sorting, rule evaluation, persistence, runtime publishing, actions, and dialogs unchanged.
- Update Monitor widget documentation/help where the LED-based body description is currently documented.

## Starting Point
- `MonitorControl.axaml` currently renders:
  - LED `Border` in the first column
  - `EventId`
  - `EventText`
  - cog action `Button`
- `MonitorRuleRow` currently exposes:
  - `RowBackground => _ownerItem.EffectiveBodyBackground`
  - `RowBorderBrush => _ownerItem.EffectiveBodyBorder`
  - `LedBrush`, `InactiveLedBrush`, `EffectiveLedBrush`
  - `StatusForegroundBrush`
- Active state is currently only strongly visible through the small LED, which is too subtle in the compact layout.
- The cog action button uses `Classes="toolbar"` and a 14x14 icon, which makes the action visually heavier than intended.

## Tasks
1. Remove the dedicated LED column from the row layout.
   - Remove the LED `Border` from `MonitorControl.axaml`.
   - Adjust `Grid.ColumnDefinitions` and column assignments.
2. Make active rows visually distinct:
   - inactive row: keep the neutral body background and border.
   - active row: apply severity-colored border and subtle severity-tinted background.
   - reuse existing log severity color mapping from `MonitorRuleRow.GetSeverityForeground()`.
3. Add or adjust row visual properties in `MonitorRuleRow`:
   - `RowBackground` should return neutral background when inactive and a subtle active background when active.
   - `RowBorderBrush` should return neutral border when inactive and severity color when active.
   - optionally add `EventIdForeground` or `EventTextForeground` only if needed for readability.
4. Generate active background colors without new dependencies.
   - Prefer a small local helper that mixes the severity color with the current body background.
   - Keep the tint subtle enough for dark and light themes.
   - If color parsing fails, fall back to the neutral body background.
5. Remove unused LED properties if no longer referenced:
   - `LedBrush`
   - `InactiveLedBrush`
   - `EffectiveLedBrush`
   - only if they are not used by tests or other code.
6. Update property notifications:
   - on active-state changes, raise `RowBackground` and `RowBorderBrush`.
   - on theme changes, raise any row visual properties affected by theme or active state.
7. Make the cog action visually buttonless:
   - keep a `Button` for click/touch and accessibility.
   - remove the visible toolbar background/border by using a transparent style or inline properties.
   - keep a stable hit target around `24-28px`.
   - reduce the icon to about `12x12`.
   - keep tooltip `Rule actions: {Name}`.
8. Ensure hover/pressed can still give minimal feedback if simple with existing styles.
   - Do not spend scope on a reusable global icon-button style.
9. Update documentation:
   - replace “severity LED” wording with active row styling wording.
   - mention that active rows use severity-colored emphasis.

## File-Level Changes
- `src/HornetStudio.Editor/Widgets/Monitor/MonitorControl.axaml` -> remove LED column; adjust row grid; lighten cog action button.
- `src/HornetStudio.Editor/Widgets/Monitor/MonitorControl.axaml.cs` -> active row background/border logic; remove LED properties if unused; update notifications.
- `src/HornetStudio/docs/widgets/Monitor.md` -> update body layout documentation.
- `src/HornetStudio/docs/widgets/help/Monitor.help.md` -> update help text for active row indication.
- `src/HornetStudio/docs/widgets/descriptions/Monitor.md` -> update only if it mentions LED/status indication.

## Implementation Order
1. Inspect current `MonitorControl.axaml` and `MonitorRuleRow` visual properties.
2. Remove the LED from XAML and shift `EventId`, `EventText`, and action button columns.
3. Add active row background/border behavior in `MonitorRuleRow`.
4. Remove or leave unused LED properties based on compiler/reference feedback.
5. Update notification calls for active/theme changes.
6. Lighten the cog button:
   - transparent background/border
   - smaller icon
   - stable click target
7. Update docs/help.
8. Run focused build and editor tests.

## Technical Constraints
- Chat responses must be German; code, comments, XML documentation, UI text, file names, and technical identifiers must remain English.
- Do not introduce new dependencies.
- Keep UI text English.
- Keep the change visual-only and scoped to MonitorWidget rows.
- Use existing Avalonia controls and existing theme/log-level colors.
- Avoid global style changes unless strictly necessary.

## Acceptance Criteria
- Active Monitor rule rows are clearly distinguishable without relying on a small LED.
- Active row border uses the rule's log-level/severity color.
- Active row background is subtly severity-tinted and remains readable in dark and light themes.
- Inactive rows remain neutral.
- The dedicated LED is no longer shown as the primary active marker.
- Cog action icon is smaller and appears buttonless/transparent while remaining clickable/touchable.
- Row action menu still opens and `Edit`/`Delete` still operate on the correct rule.
- Monitor documentation no longer describes the body as LED-based if the LED is removed.

## Verification
- Run `dotnet build src/HornetStudio.Editor/HornetStudio.Editor.csproj --no-restore` for focused XAML/C# verification.
- Run `dotnet src/HornetStudio.Editor.Tests/bin/Debug/net9.0-windows/HornetStudio.Editor.Tests.dll`.
- If no running app/debugger locks the app output, also run `dotnet build HornetStudio.sln --no-restore`.
- Manual visual check:
  - one active Warning/Error/Fatal row is clearly visible
  - inactive rows stay visually quiet
  - cog icon is smaller and does not look like a heavy button
  - row action menu remains touch/click accessible

## Out of Scope
- Changing rule sorting, EventId behavior, aggregate metadata, runtime item shape, or rule editor dialog.
- Creating a global icon-only button style.
- Adding animation, badges, counters, grouping headers, or filters.
- Removing the row action menu.

## Risks / Watchouts
- Too strong active background can reduce text contrast. Keep tint subtle and border stronger.
- Too transparent action button may become undiscoverable. Keep the cog icon visible and tooltip intact.
- Inline transparent button styling must not break hit testing or context menu opening.
- If the app is running, full solution build may fail due locked output DLLs; focused Editor build is still useful for the changed files.
