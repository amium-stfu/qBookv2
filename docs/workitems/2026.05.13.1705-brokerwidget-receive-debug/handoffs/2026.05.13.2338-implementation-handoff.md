# IMPLEMENTATION HANDOFF

## Goal
Fix the ItemClient/EnhancedSignal write feedback loop where changing an original value from `10000` to `100` causes the value to oscillate between `1000` and `100`.

## Starting Point
- UI write diagnostics already showed the UI path can update the local `write` property.
- `DataRegistry.UpdateProperty(...)` now supports `forceChangeNotification`, and UI writes to `write` use it.
- `HostItemBrokerWriteBackClient` applies `ItemWriteRequestMessage`, but its write-request path currently uses `UpdateProperty(..., "write", value)` without `forceChangeNotification`.
- `EnhancedSignalRuntime.ForwardSetInput(...)` intentionally applies inverse adjustment before writing the source target, then calls `UpdateLocalSetInput(...)` with the original enhanced-signal input value.
- `MqttItemClientSession` treats non-retained `.../write` messages as `ItemWriteRequestMessage`, but retained `.../write` messages are applied to received state as normal property changes.
- ItemClient snapshots can publish `write` properties as retained snapshot state.

Current working hypothesis:
- `100` is the enhanced-signal/UI-domain write value.
- `1000` is the inverse-adjusted source-domain write value.
- Both values can be reintroduced into the same local write path through retained snapshot/property publish and write-back subscriptions, causing a feedback loop.

## Risks / Watchouts
- MQTT retain semantics matter: a retained `write` topic can look like state, while a non-retained `write` topic is a command in the current external Item client.
- Source-domain and enhanced-domain values can differ by adjustment/inverse adjustment. Treating both as the same value path creates loops.
- Suppressing all `write` publishes would break legitimate command writes. The fix must distinguish retained snapshot state from live command publication.
- `sourceClientId` is usually unavailable with the current topic mapper, so echo detection cannot rely only on client id.
- `IncludeRetained` on `ItemSubscriptionOptions` may not be enforced by the external client subscription implementation, so HornetStudio may need its own retained/state guard.
