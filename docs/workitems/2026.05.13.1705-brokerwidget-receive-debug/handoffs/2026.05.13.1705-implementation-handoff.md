## Important Finding
`BrokerWidget` does not currently use `MqttItemClient.ReceivedItems` directly.

Current receive path:

```text
BrokerWidget
  -> IHostItemBrokerClient.GetItemSnapshots()
    -> HostItemBrokerClient.Items
      -> MqttItemClient.GetRemoteItemSnapshots()
        -> MqttReceivedItemRegistry.GetItemRoots()
```

`MqttReceivedItemRegistry` is only used indirectly through `GetRemoteItemSnapshots()`.

This matters because `MqttItemClient.GetRemoteItemSnapshots()` is a compatibility/visibility layer. It filters:
- locally published paths in `_publishedServerPaths`
- descendants of locally published paths
- roots without visible content

Therefore, a failure can be:
- receive/subscription/mapping failure before `ReceivedItems`, or
- filtering/visibility failure after `ReceivedItems`.

```text

```

Recommended handling:
- Downgrade or gate high-frequency successful publish logs.
- Keep warnings/errors.
- Keep connect/subscribe/receive diagnostics compact and signature-based.

## Risks / Watchouts
- `GetRemoteItemSnapshots()` hides paths that this same client published. Since the current widget both publishes and receives, self-published path filtering can accidentally hide expected data if paths overlap.
- With `BrokerBaseTopic=edm1`, received item paths are relative to that base topic, so attach identities will not include `edm1`.
- With empty `BrokerBaseTopic`, received item paths should include `edm1`.
- The external Item solution may be newer than the NuGet package currently referenced by HornetStudio. HornetStudio references `Amium.Item.Client` package versions in `.csproj`; do not assume local source behavior is active at runtime unless the package version matches.
- `MqttItemClientSession` receive diagnostics are guarded by `AppContext` switch `Amium.Item.Server.Mqtt.ReceiveDiagnostics`; enabling it may help but should not be required for normal operation.
- Do not confuse write-back subscriptions (`SubscribeAsync`) with received-item registry reconstruction. They are related but not the same path.
