# IMPLEMENTATION HANDOFF

## Goal
Update the function workflow step editor so each configured step is rendered as one compact, table-like row with identical column widths, no redundant summary text, theme-compliant fields, and backgroundless icon buttons.

## Scope
Included:
- Rework the configured step row layout in `WorkflowEditorDialogWindow.axaml`.
- Remove the redundant per-step summary row/text from normal editable rows.
- Render `Delay`, `Log`, and `SetValue` as a single horizontal row.
- Keep all step rows on a shared fixed column model so fields align across step types.
- Place the argument/target list button directly to the right of the argument/target field.
- Replace the `SetValue` text `Browse` button with a `List.svg` icon button.
- Prepare a second `List.svg` icon button for future value-from-item selection where the SetValue row needs it.
- Render `Delay` as value plus unit only, for example `3000` and `ms`.
- Make icon-only buttons visually backgroundless in their normal state while keeping theme-driven icon tint and accessible hover/focus behavior.

Explicitly not included:
- No new business behavior for passing a value from another item.
- No workflow serialization or execution changes.
- No new dependencies.
- No unrelated refactoring.

## Starting Point
- Current row template is in `src/HornetStudio.Editor/Widgets/Workflow/WorkflowEditorDialogWindow.axaml`.
- Current editable step rows show a first line with the step type and a summary, then a second line with actual fields.
- This two-line structure is explicitly unwanted.
- Current icon buttons still show a filled button background/border in normal state.
- Current `SetValue` layout places icon/list buttons in a visually ambiguous middle area.
- Current `Delay` row shows an explanatory sentence; the target structure should only show the numeric value and `ms`.
- `OnBrowseStepTargetClicked` in `WorkflowEditorDialogWindow.axaml.cs` already provides target selection behavior and should stay connected to the replacement target list icon.

## Tasks
1. Remove the editable-row summary display from the normal step row layout.
2. Create one shared row grid for editable steps with identical column widths.
3. Map each step type into the shared row grid without adding a second field row.
4. Place the argument/target field and its list icon as adjacent columns.
5. Replace `Browse` text with a `List.svg` `ThemeSvgIcon` button.
6. Add the prepared value-reference `List.svg` button beside the value-related area only as UI preparation.
7. Change icon button styling so normal-state icon buttons have transparent background and border.
8. Keep text input and combo box styling theme-bound with visible field borders.
9. Preserve nested `IfThenElse` block structure while removing unnecessary summary text from its editable child rows.
