# IMPLEMENTATION HANDOFF

## Goal
Update the function workflow step editor layout so configured steps use a calm, table-like, left-aligned structure with identical column widths across step types while staying compliant with existing HornetStudio theme rules.

## Scope
Included:
- Adjust the configured step row layout in the function editor dialog.
- Use one shared column model for `Delay`, `Log`, `SetValue`, and `IfThenElse` step rows.
- Keep input fields and combo boxes left-aligned and visually consistent with the SVG target layout.
- Replace the `SetValue` text `Browse` button with a backgroundless icon button using `List.svg`.
- Prepare an additional backgroundless `List.svg` icon button beside the argument/target field for a future item-value reference feature.
- Keep existing behavior for target browsing.

Explicitly not included:
- Implementing the future item-value reference behavior.
- Adding new runtime dependencies.
- Refactoring workflow execution, serialization, or function definition model behavior.
- Changing public file format semantics.

## Starting Point
- Current configured step rows are defined primarily in `src/HornetStudio.Editor/Widgets/Workflow/WorkflowEditorDialogWindow.axaml`.
- The current row template uses separate grids with different column definitions per step type:
  - `SetValue`: `*,Auto,*`
  - `Delay`: `220,*`
  - `Log`: `280,140,*`
- This creates uneven left edges and an unstable visual rhythm.
- Icon-style buttons currently use `Button.icon-button`, but the style still binds `Background` and `BorderBrush` to button theme values, creating a visible button surface.
- `OnBrowseStepTargetClicked` in `WorkflowEditorDialogWindow.axaml.cs` already implements the target picker behavior and should remain the behavior behind the replacement icon button.
- A repository search did not find a committed `List.svg` asset. The implementation must locate the intended asset if it exists under another name/path or add it in the existing icon asset location using the project's existing SVG icon pattern.

## Tasks
1. Inspect existing icon asset conventions under the editor icon folders and confirm the correct location/path for `List.svg`.
2. Define one shared configured-step column model for all row detail grids.
3. Update `FunctionStepRowTemplate` so all step detail fields use identical column widths and left-aligned field starts.
4. Replace the `SetValue` `Browse` text button with a backgroundless icon button using `List.svg`, keeping `OnBrowseStepTargetClicked`.
5. Add the prepared backgroundless `List.svg` icon button beside the argument/target field without implementing new item-value reference behavior.
6. Adjust icon button styling so pure icon buttons have no visible background while preserving theme-driven icon tinting and interaction states.
7. Keep text boxes and combo boxes theme-bound and framed according to the current dialog theme resources.
8. Verify the dialog still builds and the XAML remains valid.

## File-Level Changes
- `src/HornetStudio.Editor/Widgets/Workflow/WorkflowEditorDialogWindow.axaml` -> Main layout changes, shared step grid, icon button replacement, field alignment, button style adjustments.
- `src/HornetStudio.Editor/Widgets/Workflow/WorkflowEditorDialogWindow.axaml.cs` -> Only change if a dedicated placeholder event handler is unavoidable for the new prepared argument button; otherwise keep unchanged.
- `src/HornetStudio.Editor/EditorIcons/List.svg` or the existing equivalent icon path -> Add or reuse the list icon asset only if it is missing from the repository.

## Implementation Order
1. Confirm the project icon asset path and whether `List.svg` already exists under a different casing or folder.
2. Introduce the shared column layout in the row template with fixed, identical widths for step type, argument/target, argument icon, modifier, value, and row action columns.
3. Apply that grid to `Delay`, `Log`, `SetValue`, and `IfThenElse` detail rows.
4. Convert the `SetValue` browse action from text button to icon button.
5. Add the prepared argument-side list icon button as a non-functional UI placeholder or disabled control if no existing command pattern supports a no-op placeholder.
6. Update icon button style to remove visible background/border while keeping theme tint and usable hover/focus feedback according to existing theme rules.
7. Run formatting/build validation.

## Technical Constraints
- Chat responses must remain German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must remain English.
- Use existing Avalonia patterns and existing project controls such as `controls:ThemeSvgIcon`.
- Do not introduce new dependencies.
- Do not hard-code new fixed colors when an existing theme binding/resource is available.

- Keep changes limited to the workflow/function editor layout scope.
- Prefer named parameters for any C# method calls if C# changes become necessary.

## Acceptance Criteria
- Configured step fields align in a table-like layout with identical column widths across all step types.
- Field left edges are stable for comparable fields in `Delay`, `Log`, `SetValue`, and nested `IfThenElse` rows.
- Combo boxes and text boxes keep a framed theme-compliant look.
- Pure icon action buttons render without a visible button background.
- `SetValue` no longer shows a text `Browse` button and instead shows the list icon.
- A second list icon button is visible beside the argument/target field as preparation for later item-value passing.
- Existing target browse behavior still works from the replacement icon button.
- No new business behavior is added for the prepared argument-side button.
