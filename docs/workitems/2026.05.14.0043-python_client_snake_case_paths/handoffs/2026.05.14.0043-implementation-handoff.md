# IMPLEMENTATION HANDOFF

## Goal
Fix PythonClient and ApplicationExplorer registry path generation so Python application values are published only under valid lowercase snake_case item paths.

## Scope
Included:
- Normalize PythonClient registry paths to strict lowercase snake_case before writing to `HostRegistries.Data`.
- Replace ApplicationExplorer-generated `Applications.Python` registry segments with canonical `applications.python`.
- Preserve existing ApplicationExplorer interaction target behavior unless a path is written into the item registry.
- Update Python path examples in source-of-truth Python documentation and copied template documentation.
- Add focused regression coverage for Python value registry path normalization.

Not included:
- Changing the on-disk folder layout `Applications/Python`.
- Renaming the persisted widget type `ApplicationExplorer`.
- Redesigning Python interaction target identifiers.
- Adding dependencies.
- Release/version changes.

## Starting Point
- `PythonClient.NormalizeRegistryPath(...)` currently splits path delimiters and calls `SanitizePathSegment(...)`.
- `SanitizePathSegment(...)` preserves casing, so values such as `Applications`, `PythonClients`, and `Raw` can reach the strict item path validator.
- `ApplicationEntryRow.BuildValueRegistryRootPath(...)` currently returns `studio.{folderName}.Applications.Python.{envName}`.
- The thrown exception is `System.ArgumentException: Item path segment 'Applications' must use lowercase snake_case and start with a letter. (Parameter 'path')`.
- Host-side canonical snake_case normalization already exists as `HostPathSegmentNormalizer.Normalize(...)`.

## Tasks
1. Inspect every PythonClient registry write path:
   - `HandleDefineValue(...)`
   - `HandleValueUpdate(...)`
   - `HandleHostValueWrite(...)`
   - `BuildValueSnapshot(...)`
2. Replace PythonClient's path-segment sanitizing behavior with strict snake_case normalization.
3. Ensure PythonClient fallback paths use lowercase fixed segments:
   - `python_clients`
4. Change ApplicationExplorer Python value registry root generation to canonical segments:
   - `studio.<folder>.applications.python.<env>`
5. Normalize dynamic folder and environment segments before composing the registry root.
6. Check legacy/current script-value publishing in `PageItemModel` for the same `Applications.Python` registry-path pattern and align it if it writes into `HostRegistries.Data`.
7. Update Python documentation examples that mention old mixed-case registry paths.
8. Add focused tests that prove mixed-case Python client names and value names produce valid snake_case registry paths.
9. Run focused build/test verification.

## Acceptance Criteria

- Existing generated Python application folders still live under `Applications/Python`.
- Documentation examples no longer show mixed-case registry paths.
