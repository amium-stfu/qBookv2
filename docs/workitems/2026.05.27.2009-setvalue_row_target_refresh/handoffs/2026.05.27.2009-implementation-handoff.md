# IMPLEMENTATION HANDOFF

## Goal
Fix the Interaction Rules SetValue editor so the Edit SetValue dialog always opens with the currently edited row target and resolves the target type from that target.

## Scope
Included:
- Configured Interaction Rules rows using action `SetValue`.
- The Edit SetValue button next to the configured row argument summary.
- Target descriptor refresh when opening the dialog.
- Focused regression coverage for row target handoff and type resolution.

Not included:
- Redesigning the SetValue operation model.
- Changing UDL runtime metadata generation beyond what is required for this bug.
- Changing New-row SetValue behavior unless a shared helper is touched and must stay consistent.

## Starting Point
- A configured row displays target `udl1.m001.set`.
- Opening Edit SetValue can show `Target = this` and `Detected type = Unknown`.
- The expected behavior is that the dialog shows `Target = udl1.m001.set` and detects the configured `type = float` as numeric.
- `SimulatedHostUdlClient` demo channels now publish `type` metadata, but the dialog still appears to receive or resolve the wrong target in this scenario.

## Tasks
1. Trace the configured-row Edit SetValue path from the cog button to `SetValueOperationEditorDialogWindow`.
2. Ensure the row edit handler uses the current `ItemInteractionEditorRow.TargetPath` as the source of truth.
3. Ensure descriptor fallback preserves the requested row target path instead of replacing it with `this`.
4. Refresh or resolve target descriptor data at dialog-open time, not only when the dialog/window was first built.
5. Add focused tests that cover a configured SetValue row with target `udl1.m001.set` and `type = float`.
6. Verify the dialog receives the row target and classifies the target as numeric.

## File-Level Changes
- `src/HornetStudio.Editor/Widgets/Common/InteractionRulesEditorDialogWindow.axaml.cs` -> verify/fix configured-row Edit SetValue click handling and descriptor construction.
- `src/HornetStudio.Editor/ViewModels/EditorDialogField.cs` -> verify/fix `GetSetValueTargetDescriptor` and descriptor rebuild behavior so current registry metadata is used and the requested target path is preserved.
- `src/HornetStudio.Editor/ViewModels/ItemInteractionEditorRow.cs` -> only if row property change notification is missing for target/SetValue metadata refresh.
- `src/HornetStudio.Editor/Widgets/Common/SetValueOperationEditorDialogWindow.axaml.cs` -> only if constructor/default handling can overwrite the passed target path.
- `src/HornetStudio.Editor.Tests/Program.cs` -> add or update focused regression tests.

## Implementation Order
1. Inspect `OnEditRowSetValueClicked` and confirm which value is passed into `GetSetValueTargetDescriptor` and the dialog constructor.
2. Add a minimal test around the row edit target handoff or the descriptor resolution method before changing behavior where practical.
3. Fix the row edit path so `row.TargetPath` is normalized once and passed through consistently.
4. Fix descriptor fallback so unresolved targets display the requested target path, not `this`.
5. If `_interactionSetValueTargets` is stale, rebuild or bypass stale cache at dialog-open time for the requested target.
6. Run focused build/tests and document any unrelated pre-existing failures.

## Technical Constraints
- Follow existing Avalonia MVVM/event-handler patterns in the dialog.
- Keep UI text and validation messages in English.
- Do not introduce new dependencies.
- Keep the change narrow; avoid broad Interaction Rules refactoring.
- Preserve existing behavior for RunFunction, Python function, dialog actions, and New-row SetValue editing.

## Acceptance Criteria
- Clicking the cog on a configured SetValue row with target `udl1.m001.set` opens Edit SetValue with `Target = udl1.m001.set`.
- The dialog detects `type = float` as `Numeric`.
- If the target cannot be resolved, the dialog still displays the requested target path and only the detected type falls back to `Unknown`.
- The configured row summary remains bound to the row argument and updates after saving the dialog.
- No regression in New-row SetValue editing.

## Out of Scope
- Full type metadata policy for all item producers.
- Item picker redesign.
- Additional SetValue operations beyond preserving the current dialog behavior.
- Documentation updates unless user-visible behavior changes beyond this bug fix.

## Risks / Watchouts
- The target options/descriptors may be cached when the Interaction Rules dialog opens, while UDL metadata can arrive later.
- A fallback path that normalizes empty values to `this` must not run for non-empty row targets.
- The configured-row and New-row handlers look similar; avoid fixing one by accidentally changing the other.
- Current full console test run may already have unrelated failures; report them separately if they remain.
