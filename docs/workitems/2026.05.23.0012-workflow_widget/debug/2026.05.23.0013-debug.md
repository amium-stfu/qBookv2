# DEBUG REPORT

## Problem
The `WorkflowWidget` appeared as an empty selected rectangle in the folder editor. After adding a workflow, the UI briefly froze and the newly placed widget disappeared, but placing another `WorkflowWidget` showed the created workflow file.

## Expected Behavior
The folder editor should render the same `WorkflowControl` used by the page editor, including header actions and body content. Adding a workflow should keep the current widget visible and create the workflow YAML file.

## Actual Behavior
Only the folder editor selection adorners were visible before the first fix. After that, adding a workflow created the file but the currently placed widget disappeared until it was placed again.

## Error Messages / Logs
No runtime or build error was observed.

## Relevant Code
- `src/HornetStudio.Editor/Controls/PageEditorControl.axaml` already renders `widgets:WorkflowControl` for `IsWorkflowWidget`.
- `src/HornetStudio.Editor/Widgets/FolderEditor/FolderEditorControl.axaml` did not render `controls:WorkflowControl` for `IsWorkflowWidget`.
- `src/HornetStudio.Editor/Widgets/Workflow/WorkflowControl.axaml.cs` created the workflow YAML file without first persisting the owning page/folder layout.
- `src/HornetStudio/ViewModels/MainWindowViewModel.cs` watched `*.yaml` recursively below `Folders`, so `Scripts/Workflows/*.yaml` was treated like a folder layout YAML and triggered a full folder reload.

## Attempted Fixes
- Added `WorkflowControl` to the folder editor item template after `MonitorControl`.
- Persisted the owning page YAML before creating a workflow YAML file, so a folder reload triggered by the new file can restore the just-placed widget.
- Marked `Add` and `Refresh` click events as handled immediately.
- Filtered book watcher events so only `Folder.yaml` and legacy `Page.yaml` trigger watched-page reloads. Workflow YAML files are now ignored by the folder-layout watcher and should be refreshed only by the `WorkflowWidget`.

## Current Hypothesis
- The first blank widget was caused by incomplete widget registration in the active folder editor XAML path.
- The disappearance after `Add` was caused by the recursive `*.yaml` watcher treating `Scripts/Workflows/*.yaml` as a folder layout change and rebuilding the complete folder list.
