# DEBUG REPORT

## Problem
The Interaction Rules `Argument` column for `SetValue` was too narrow even though the dialog had unused horizontal space. The `Edit SetValue` dialog also detected `EnhancedSignal.Set` targets as `Unknown`.

## Expected Behavior
The `Argument` editor should use the available row width. A target such as `enhanced_signals.filtered_1.set` should be detected as numeric when its sibling `read` channel currently carries a numeric value.

## Actual Behavior
The row grid used a fixed 150 px `Argument` column. The editor target descriptor resolved relative targets without the owner folder context and only inspected the selected `set` item itself, whose value can be empty before the first write.

## Error Messages / Logs
No runtime exception. The issue was visible in the editor UI as a narrow summary field and `Detected type: Unknown`.

## Relevant Code
- `src/HornetStudio.Editor/Widgets/Common/InteractionRulesEditorDialogWindow.axaml`
- `src/HornetStudio.Editor/ViewModels/EditorDialogField.cs`
- `src/HornetStudio.Editor/Models/PageItemModel.cs`

## Attempted Fixes
- Fix 1 -> Changed the Interaction Rules row grids from a fixed 150 px `Argument` column to a star-sized column, so the summary/text field consumes remaining width. Result: focused editor build succeeds.
- Fix 2 -> Changed editor target descriptor resolution to use `OwnerItem.FolderName`, matching runtime target resolution for relative paths. Result: focused editor build succeeds.
- Fix 3 -> Added `.set` sibling `.read` value fallback for target kind/current-value detection in both editor metadata and runtime `SetValue` operation resolution. Result: focused editor build succeeds.

## Current Hypothesis
- `EnhancedSignal.Set` should now classify from the corresponding `EnhancedSignal.Read` value when the set channel has not yet received a value.
